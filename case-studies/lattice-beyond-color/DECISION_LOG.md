# Decision Log — Lattice Beyond Color

## Decision 001 — Use function before metaphor

**Choice:** Begin with the failure mode, not the organism or material.

**Reason:** Starting with a visual reference encourages decorative biomimicry. Starting with a failure worth solving allows the source mechanism to be tested for causal relevance.

**Hierarchy:** failure mode → physical mechanism → measurable model → user decision → interface geometry → material treatment → narrative.

## Decision 002 — Preserve one compiler, permit multiple topologies

**Invariant:** evidence → mechanism → formula → semantic parameter → behavior → visible state → verification.

**Variable:** lattice topology, propagation medium, failure mode, interaction, material language, and commercial narrative.

**Reason:** Schema-level sameness creates interoperability. Topological diversity prevents five differently colored versions of the same dashboard.

## Decision 003 — Deterministic core, probabilistic edge

Deterministic responsibilities:

- evidence IDs and provenance
- normalization and parameter serialization
- formula and policy versions
- state transitions
- rendering from serialized state
- audit history and replay

Probabilistic responsibilities:

- forecasts
- source or threat classification
- inferred relationships
- demand scenarios
- candidate actions with confidence and expiry

**Gate:** probability may propose; it may not silently rewrite provenance or execute irreversible policy.

## Decision 004 — Dual-encode critical states

Color alone is never sufficient. Every critical state must also alter one or more of geometry, aperture, width, discontinuity, roughness, hatch, label, vibration, or motion timing.

## Decision 005 — Bound generative divergence

All procedural variation accepts a stored seed and serializable parameters. Production divergence remains within a 35% budget unless evidence explicitly justifies a larger departure.

## Decision 006 — Distinguish imagination from innovation

- **Imagination:** generates diverse possibilities upstream.
- **Innovation:** survives relevance, implementation, adoption, and verification downstream.

A novel appearance without a new causal capability remains imaginative, not innovative.

## Concept choices

### HelioVeil

Selected because photonic spacing, aperture, and radiative behavior map directly to energy and comfort decisions. Rejected as a purely iridescent façade treatment.

### EchoVault

Selected because phononic geometry changes frequency propagation and spatial privacy. Rejected as a decorative audio visualizer or standard equalizer reskin.

### Fault Garden

Selected because crystal defect classes provide a rigorous semantic grammar for missing, conflicting, imported, and incompatible evidence. This is the flagship software direction because it exercises inference, provenance, and replay without hardware dependency.

### Capillary Commons

Selected because reticulate venation expresses redundancy and service preservation under failure. Rejected if the organic network cannot outperform or explain a cheaper tree topology.

### Nacre Sentinel

Selected because staggered layers, sacrificial interfaces, bridging, and crack deflection translate cleanly into containment architecture. This is the strongest enterprise selling proof because the failure path is legible to both technical and executive audiences.

## Explainability contract

Every visible state must be reconstructable as:

```text
source evidence
+ formula or rule version
+ normalized inputs
+ seed, when generative
+ derived semantic parameters
+ policy gate
= rendered state and action
```

A state that cannot be replayed from this receipt fails the system definition of done.
