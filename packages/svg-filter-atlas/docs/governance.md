# Filter governance and approval

## Statuses

- `draft`: incomplete and not available to production consumers.
- `experimental`: useful for exploration but not performance- or browser-approved.
- `approved`: validated, documented, and safe within its stated limits.
- `deprecated`: retained for compatibility but excluded from new work.

## Approval gate

A recipe is approved only when these representations agree:

1. Intent: the material or visual behavior being pursued.
2. Specification: primitives, parameters, bounds, and presets.
3. Implementation: generated SVG and framework output.
4. Observable behavior: screenshots across supported engines.
5. Evidence: validation, regression, accessibility, and performance records.

## Screen-level rule

Prefer one foundational material, one depth or light treatment, and one accent effect. More than one heavy optical filter on the same element requires an explicit exception.
