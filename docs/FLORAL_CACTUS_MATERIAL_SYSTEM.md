# Seed-Loom Floral + Cactus Material System

## Purpose

The floral and cactus layer is not decorative garnish. It is a functional symbolic, spatial, motion, and state system inherited from Omni-Loom’s tactile fabrication language and adapted for Seed-Loom’s traceable signal-to-system workflow.

The visual thesis is:

> A living system can be soft, thorned, layered, resilient, and still remain exact.

## Functional symbolism

| Form | System meaning | UI use |
|---|---|---|
| Seed | source signal, latent possibility | intake, new run, empty state |
| Root | provenance, hidden dependency | trace paths, lineage, source graph |
| Stem | active transformation | pipeline progress, vertical navigation |
| Branch | divergence or alternate interpretation | disambiguation, variants, scenario forks |
| Leaf | reusable output | token, component, export, derived artifact |
| Bud | unresolved but promising inference | candidate state, review queue |
| Flower | validated synthesis | completed run, featured system, commercial pack |
| Fruit | productized value | marketplace item, licensed output |
| Cactus body | durable system boundary | workspace, reference package, protected invariant |
| Rib | repeated structural rule | grid, schema field group, parameter family |
| Spine | warning, constraint, or refusal | validation error, anti-pattern, forbidden change |
| Areole | attachment point | relation node, evidence link, plugin connection |
| Blooming cactus | resilient completion under constraint | flagship proof, verified launch state |

## Composition rules

1. Flora grows from data-bearing edges, seams, source lines, cards, and graph nodes—never from arbitrary corners without a relationship.
2. Cacti occupy structurally important zones: navigation anchors, evidence columns, protected references, pricing boundaries, and system status landmarks.
3. Flowers appear at synthesis or conversion moments, not continuously.
4. Spines must correspond to a real constraint, risk, failure mode, or validation rule.
5. Roots and threads may share a visual grammar but remain semantically distinct: roots indicate lineage; threads indicate an explicit traceable translation.
6. A screen may contain one dominant botanical organism and up to three supporting fragments. Avoid wallpaper repetition.
7. Botanical forms must preserve utility: no petals behind body copy, no spines near primary tap targets, and no texture reducing code readability.

## Material model

### Botanical layers

- pressed paper petals
- embroidered vein lines
- quilted leaf bodies
- translucent vellum membranes
- hammered copper stamens
- glazed ceramic cactus ribs
- velvet or flocked shadow planes
- woven linen ground
- iridescent filament highlights

### Depth stack

1. ground weave
2. printed pigment
3. embossed or quilted volume
4. stitched contour
5. metallic attachment
6. translucent highlight
7. cast shadow
8. interactive glow or motion response

Every advanced component should expose at least three of these layers. Hero and flagship moments should expose five or more.

## 3D behavior

### Parallax

- background weave: 0.15x pointer displacement
- botanical silhouette: 0.35x
- leaf and petal body: 0.6x
- metallic detail: 0.85x
- pollen, thread, or spine highlight: 1.0x

Maximum apparent rotation is 5 degrees for utility components and 9 degrees for hero objects.

### Light response

- linen responds diffusely
- satin and vellum use broad anisotropic highlights
- copper uses a narrow warm highlight
- ceramic cactus surfaces use a soft glossy ridge highlight
- stitches remain matte and should not bloom excessively

### Compression

Pressed cards and buttons reduce depth by 35–55%, tighten shadow spread, and pull stitched borders inward by 1–2px.

## Motion grammar

| Motion | Duration | Meaning |
|---|---:|---|
| germinate | 620ms | a source becomes an active run |
| root-trace | 520ms | provenance is revealed |
| branch-fork | 420ms | alternatives become visible |
| leaf-unfurl | 480ms | a reusable artifact is exposed |
| bloom-validate | 760ms | synthesis passes verification |
| spine-alert | 180ms | a constraint interrupts progress |
| cactus-breathe | 4200ms | durable system remains alive |
| pollen-drift | 6000–9000ms | ambient noncritical activity |
| fabric-settle | 760ms | a tactile layer gains weight |

Reduced motion replaces growth transforms with opacity, label changes, persistent trace lines, and static expanded states.

## Component upgrades

### Hero organism

A sculptural woven cactus-flower hybrid acts as the central interactive object. Its ribs correspond to the five translation stages: evidence, interpretation, tokens, code, and verification. Selecting a stage lights one rib and reveals a matching panel.

### Analyzer source field

The upload zone becomes a layered specimen table with paper, fabric, ceramic, and stitched boundaries. Source regions appear as translucent botanical cells. Confidence is represented by leaf saturation and edge integrity, not color alone.

### Interpretation cards

Primary interpretation is a full flower. Alternatives appear as buds on branching stems. Unsupported claims display clipped or frayed petals. A disambiguation question opens the branch rather than replacing the prior state.

### Token field

Tokens become specimen tags attached to leaf, spine, petal, rib, or material samples. Each tag exposes source evidence, confidence, inheritance, target code, and allowed variance.

### Code translation field

Code remains on a flat high-contrast inset surface. Botanical forms frame the code rather than appearing behind it. Hovering a variable illuminates the corresponding material sample and trace line.

### Commerce field

Plans should represent stages of cultivation rather than generic SaaS tiers:

- Seed / explore
- Garden / studio
- Conservatory / pro
- Biome / enterprise

Product cards show the organism, its outputs, its license boundary, and its verified translation targets.

## Anti-patterns

- generic flat floral SVG decoration
- photorealistic stock cactus imagery
- tropical branding unrelated to the system
- flowers on every card
- texture applied as a low-opacity noise overlay only
- parallax without semantic relationship
- gradients replacing material logic
- thorns used only to look edgy
- motion that makes the analyzer harder to read
- 3D depth without press, focus, reduced-motion, and keyboard states

## Acceptance criteria

A screen passes when:

- each botanical form has a named semantic role
- all significant visual layers map to tokens
- depth survives without reducing legibility
- component state changes alter material behavior, not only color
- motion has a defined meaning and reduced-motion equivalent
- traceability remains visible through roots, threads, tags, or attachment points
- floral density is deliberately controlled
- mobile layouts preserve the dominant organism without obstructing controls
- no critical information is encoded solely through color, depth, or animation
